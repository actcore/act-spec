### Task 1: Update WIT file

**Files:**
- Modify: `wit/act-core.wit`

**Step 1: Replace entire WIT file with v0.1.2**

Replace contents of `wit/act-core.wit` with the complete WIT from the design doc. Key changes:
- Package version `0.1.1` → `0.1.2`
- Add `type metadata = list<tuple<string, list<u8>>>`
- All `extensions: list<tuple<string, string>>` → `metadata: metadata`
- Remove `enum error-kind` — `tool-error.kind` becomes `string`
- Remove `record tool-annotations` — `description` moves to `tool-definition`, rest becomes `metadata`
- Remove `tool-annotations` from `tool-definition`, add `description: localized-string` and `metadata: metadata`
- Add `record call-response { metadata, body }` and `record list-tools-response { metadata, tools }`
- `call-tool` returns `call-response` (not `result<stream, tool-error>`)
- `list-tools` returns `result<list-tools-response, tool-error>` (not `list<tool-definition>`)
- Update `use` imports in `tool-provider` to include new types

```wit
package act:core@0.1.2;

interface types {
  type localized-string = list<tuple<string, string>>;
  type metadata = list<tuple<string, list<u8>>>;

  record capability {
    id: string,
    description: option<localized-string>,
    metadata: metadata,
  }

  record component-capabilities {
    required: list<capability>,
    optional: list<capability>,
    metadata: metadata,
  }

  record component-info {
    name: string,
    version: string,
    default-language: string,
    description: localized-string,
    capabilities: component-capabilities,
    metadata: metadata,
  }

  record parameter-meta {
    name: string,
    description: localized-string,
    schema: string,
    required: bool,
    metadata: metadata,
  }

  record tool-definition {
    name: string,
    description: localized-string,
    parameters: list<parameter-meta>,
    metadata: metadata,
  }

  record tool-call {
    id: string,
    name: string,
    arguments: list<u8>,
    metadata: metadata,
  }

  record content-part {
    data: list<u8>,
    mime-type: option<string>,
    metadata: metadata,
  }

  record tool-error {
    kind: string,
    message: localized-string,
    metadata: metadata,
  }

  variant stream-event {
    content(content-part),
    error(tool-error),
  }

  record call-response {
    metadata: metadata,
    body: result<stream<stream-event>, tool-error>,
  }

  record list-tools-response {
    metadata: metadata,
    tools: list<tool-definition>,
  }
}

interface component-metadata {
  use types.{ component-info };
  get-info: func() -> component-info;
}

interface tool-provider {
  use types.{
    tool-definition,
    tool-call,
    call-response,
    list-tools-response,
    tool-error,
  };

  get-config-schema: func() -> option<string>;
  list-tools: func(config: option<list<u8>>) -> result<list-tools-response, tool-error>;
  call-tool: async func(config: option<list<u8>>, call: tool-call) -> call-response;
}

world act-world {
  export component-metadata;
  export tool-provider;
}
```

### Task 3: Update ACT-SPEC.md — error handling and well-known keys

**Files:**
- Modify: `spec/v0.1/ACT-SPEC.md`

**Step 1: Rewrite Section 9 (Error Handling)**

- Update two-level error model prose for new `call-response` structure (body is `result`, metadata always available)
- Replace error-kind enum reference with well-known error kind strings table:

| Kind string | Typical source | Description |
|---|---|---|
| `not-found` | Host or Component | The named tool does not exist |
| `invalid-args` | Host | Arguments or config failed schema validation |
| `timeout` | Host | Call exceeded timeout |
| `capability-denied` | Host | Component used ungranted capability |
| `internal` | Component | Unrecoverable internal error |

- Add note: custom error kinds use namespaced strings (e.g. `acme:rate-limited`). Hosts MUST NOT reject unknown error kinds.

**Step 2: Add new Section — Well-Known Metadata Keys**

Add section after Extensibility (Section 8) or as part of it. Document well-known metadata keys by context:

**Tool metadata** (in `tool-definition.metadata`):

| Key | dCBOR type | Description |
|---|---|---|
| `read-only` | bool | Tool does not modify state |
| `idempotent` | bool | Repeated calls produce same result |
| `destructive` | bool | Tool may irreversibly modify state |
| `usage-hints` | localized-string | When to use this tool |
| `anti-usage-hints` | localized-string | When NOT to use this tool |
| `examples` | array of bstr | Example calls as dCBOR |
| `tags` | array of tstr | Categorization tags |
| `timeout-ms` | uint | Suggested timeout in milliseconds |

**Response metadata** (in `call-response.metadata`, `list-tools-response.metadata`):
- Document as open-ended, no well-known keys yet in v0.1.2. Hosts and components may define their own (e.g. `cache-ttl-ms`, `request-id`).

- Add note: all metadata keys SHOULD be namespaced to avoid collisions (e.g. `acme:priority`). Well-known keys defined in this spec use bare names. Hosts and components MUST ignore unrecognized keys.

**Step 3: Update Section 8 (Extensibility)**

- Replace `extensions` field description with `metadata` field description
- Note: `metadata` values are dCBOR-encoded, not plain strings
- Remove old `list<tuple<string, string>>` references

**Step 4: Commit**

```bash
git add spec/v0.1/ACT-SPEC.md
git commit -m "spec: well-known metadata keys, extensible error kinds for v0.1.2"
```
